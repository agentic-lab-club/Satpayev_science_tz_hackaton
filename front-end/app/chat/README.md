# AI-Ассистент - Чат-страница

## Описание

Полнофункциональный интерфейс чата для взаимодействия с AI-ассистентом по вопросам составления технических заданий.

## Структура компонентов

```
/app/chat/
├── page.tsx                    # Главная страница (root component)
├── constants.ts               # Константы и примеры запросов
└── components/
    ├── ChatHeader.tsx         # Заголовок с информацией об ассистенте
    ├── ExamplePrompts.tsx     # Примеры запросов (системные подсказки)
    ├── MessageList.tsx        # Список сообщений с анимацией
    └── ChatInput.tsx          # Поле ввода + кнопка очистки
```

## Компоненты

### `page.tsx` (Chat Page)
Главная страница, управляет состояниями:
- `messages` - история сообщений
- `input` - текст в поле ввода
- `isLoading` - статус загрузки ответа
- Обработка отправки сообщений
- Имитация ответа ассистента (с задержкой)

**Функции:**
- `handleSendMessage()` - отправка сообщения
- `handleClearDialog()` - очистка диалога с подтверждением
- `handleExamplePromptSelect()` - выбор примера запроса
- Auto-scroll к новым сообщениям

### `constants.ts`
Содержит:
- `EXAMPLE_PROMPTS` - массив из 6 примеров запросов (2 категории)
  - 4 примера про правила составления ТЗ
  - 2 примера про анализ и улучшение текущего ТЗ
- `INITIAL_MESSAGE` - приветственное сообщение ассистента

**Пример запроса:**
```typescript
{
  icon: "📋",           // Эмодзи
  title: "Структурируйте ТЗ",      // Заголовок
  description: "Какие основные разделы...", // Текст для отправки
  category: "rules",    // Категория (rules или improve)
}
```

### `ChatHeader.tsx`
Липкий заголовок с информацией:
- Avatar ассистента (🤖)
- Имя "AI-Ассистент"
- Статус (онлайн / обработка запроса)
- Счетчик сообщений

### `ExamplePrompts.tsx`
Система подсказок с примерами запросов:
- 2 секции: "Правила ТЗ" и "Улучшение ТЗ"
- Каждый пример интерактивный (click → отправка)
- Hover эффекты с переходом на цвет amber
- Исчезает при первом отправленном сообщении

### `MessageList.tsx`
Список сообщений с функциями:
- Разные стили для user (amber) и assistant (slate)
- Анимация появления новых сообщений
- Индикатор печати (3 бегущие точки)
- Auto-scroll to bottom

### `ChatInput.tsx`
Интерактивное поле ввода:
- `<textarea>` с поддержкой многострочности
- `Shift+Enter` - новая строка, `Enter` - отправка
- Кнопка отправки (с состояниями: ↑ или спинер)
- **Кнопка очистки** с подтверждением удаления истории
- Счетчик символов (текущие/макс)
- Отключение кнопок при загрузке

## Стили и анимации

### Цветовая схема
- Background: `#080d14` (primary dark)
- Accent: `amber-400` (buttons, highlights)
- User messages: `amber-400/20` background
- Assistant messages: `slate-800` background
- Borders: `slate-700` (default), `amber-400/30` (hover)

### Анимации
- **Сообщения**: `animate-fade-in` (плавное появление)
- **Индикатор печати**: 3 точки с `animate-bounce`
- **Спинер**: `animate-spin` (на кнопке при отправке)
- **Sticky header**: `sticky top-0`
- **Transitions**: 200-300ms для всех hover эффектов

## Функциональность

### История сообщений
- Начинается с приветственного сообщения
- Каждое сообщение имеет тип (user/assistant)
- Стили различаются по типу
- Сохраняется только в текущей сессии

### Примеры запросов
- Показываются только при загрузке страницы (пока нет сообщений)
- 6 готовых примеров по двум категориям
- Клик на пример → отправка как обычное сообщение
- Исчезают после первого сообщения

### Очистка диалога
- Кнопка 🗑️ внизу левой части инпута
- При клике показывает подтверждение
- Удаляет всю историю
- Возвращает к исходному состоянию

### Имитация ответа
- После отправки сообщения показывается индикатор печати
- Случайная задержка ~1.5 сек
- Выбирается случайный ответ из массива `RESPONSE_SAMPLES`

## Интеграция с dashboard

Компонент `ChatWidget.tsx` из dashboard переиспользует типы:
- `ChatMessage` из `dashboard/components/types.ts`
- Могут делиться историей через контекст/API

## Планы развития

1. **API Integration** - подключить реальный бэкенд
2. **File Upload** - загрузка документа для анализа в чате
3. **Conversation Persistence** - сохранение истории в БД
4. **User Context** - переиспользование информации из dashboard
5. **Advanced Prompts** - динамические примеры на основе загруженных документов
6. **Export Chat** - экспорт истории в PDF/markdown

## Использование

### Начало чата
```tsx
// Страница автоматически загружает пример с приветствием
// Пользователь может выбрать один из примеров или написать свой вопрос
```

### Отправка сообщения
```
1. Введите текст (до 2000 символов)
2. Нажмите Enter или кнопку ↑
3. Дождитесь ответа ассистента
```

### Очистка истории
```
1. Нажмите кнопку 🗑️ Очистить диалог
2. Подтвердите действие
3. История удалена, диалог начинается заново
```
